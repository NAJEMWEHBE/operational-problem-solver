Spin up a team of agents.
Ship only the best one.

A multi-team agentic engine for Claude Code. Parallel worker teams generate competing solutions, a consensus gate filters the unstable ones, and an ELO tournament ranks the survivors head-to-head.

The pieces are proven research — AlphaCode, self-consistency, LLM-as-judge, Chatbot-Arena Elo. This packages them into one installable Claude Code skill + engine. Novel as an integration, honest about it.

One orchestrator, three crews.

Each team runs multiple instances in parallel. Workers are isolated so they don't converge on the same idea.

Quality Assurance

// execution protocol

Phase I → II → III.

If fewer than half the workers produce stable output, the engine mutates the prompts and re-loops Execution before judging.

PHASE I

Audit

Intelligence team probes the environment and builds the shared Context Manifest.

PHASE II

Generate

Execution workers produce diverse standalone solutions in parallel, each isolated.

PHASE III

Judge & ship

Consensus gate → ELO tournament → the highest-rated solution is written out.

⚡

Honest about cost. Every worker and the judge are metered Claude API calls — roughly 8–12 per run by default. Prompt caching, a --cheap Haiku tier, and fewer workers keep it down. The engine prints an estimated dollar cost every run. The offline test suite runs the ELO and consensus logic with a mocked model — zero API spend.
